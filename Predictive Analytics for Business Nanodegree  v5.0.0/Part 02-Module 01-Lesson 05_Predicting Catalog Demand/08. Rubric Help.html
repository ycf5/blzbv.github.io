08. Rubric Help

Every project has a rubric that your submission will be graded against by our reviewers. Here is the rubric for this project: https://review.udacity.com/#!/rubrics/186/view

We have noticed some areas where students are making similar mistakes and wanted to address some of the most commonly missed rubric items and give suggestions about how to how to pass them. All the data and graphs in these suggestions is different data than what is in the actual project.

Rubric Item

All following questions have been answered:

  • What decisions need to be made?
  • What data is needed to inform those decisions?

Suggestions:

When you are thinking about what decisions need to be made and what data is needed consider the following:

  • What is the company ultimately trying to make a decision on?
  • What is the criteria that will let them know what they should decide?
  • How will we predict the potential profit of sending out the catalog?
  • Considering the data we have, what pieces of it may be useful in coming up with this prediction?

Rubric item:

  • Each predictor variable is shown to have a linear relationship between the target variable whenever appropriate.
  • Each predictor variable should be significant (p-value <= 0.05)

Suggestions:

For Numeric variables we can use scatterplots between an individual variable and the target variable to see if a variable might be a good candidate for a predictor variable. Alteryx has a built in Scatterplot Tool you can use to do this. Put your target variable as your Y and a numeric predictor variable as X.

You must do this for all numeric variables

A sloped line in a scatter plot indicates that as the X increases or decreases the Y increase or decreases indicating the two variables are related. So if there is a slope to the line then this might indicate that this is a good predictor variable for this target variable. The first two graphs above would indicate the X variable (the potential predictor variable) would be a good candidate to be a predictor variable for the target variable. The last scatter plot graph shows an example of no slope and therefore there is no relationship between the predictor and target variable

For categorical variables you will have to use trial and error to see which are statistically significant. You can do this by including one in your model running it and checking your P value for the categorical variable.

I have included a screenshot from the R out put of the Linear Regression tool in Alteryx which is the Linear Regression Report below to illustrate where to check for these values.

We are looking for P-values less than 0.05 None of the categorical variables here have Pvalue that low so we would remove it and try another categorical variable. As you can see categorical variables are broken down into their individual categories in the linear regression report. If one of the individual categories is statistically significant it is a good categorical variable to consider for your model.

You may see a P-value that has a value that looks something like < 2.2e-16 this can be written as 2.2 * 10^-16 which can also be written as < .00000000000000022 This is very small! This would be a very significant variable for your model.

A shortcut to determining if the p value is statistically significant is to look out to the right of it for the Significance codes. Basically if you see one or more asterisks it is statistically significant

Rubric Item:

p-values and R-squared values are used to justify how well the linear model works.

Suggestions:

The P values and R squared values are circled in red below in the screenshot from the Report of the Linear Regression tool in Alteryx.

Low P-values and a high R-squared suggest the model is highly predictive, Low P-values means it is highly unlikely that the two variables are not related. Low R squared means the model is not very fit. Some additional reading here:

Rubric Item:

The regression equation given is correct. Each coefficient should have up to 2 digits after the decimal figures (ex: 1.28).

Suggestions:

The regression equation is built from the R output of the Linear Regression tool.
You use the coefficient estimates and variable names to create the regression equation.

I have highlighted the Coefficient Estimates in the screenshot below to show you where you will get the values to plug into the formula.

Here is the equation with our variables in place but before we have plugged in the coefficient estimates:

Target variable = intercept + coefficient * (predictorpositive) + coefficient * (If red) + coefficient * (If yellow)

Here is the equation after we have plugged in the coefficient estimates:

Target variable = 3.917+ 0.470 * (predictorpositive) + -3.484 * (If red) + -1.424 * (If yellow)

^^^This is the type of equation we are looking for in the report^^^

Side note:

The score tool takes the formula with the coefficient estimates and applies it to each row of data you pass in. So for example if the first row of the data you are applying this formula to has a 10 in the predictor positive column and has yellow in the categorical variable column then the score tool will plug those values into the formula to predict the target variable

Target variable = 3.917+ 0.470 * (10) + -3.484 * (0) + -1.424 * (1)

Target variable = 3.917 + 4.70 + 0 -1.424

Target variable = 7.193

This is what the score tool will do for each row of data you are passing in. It will create a prediction for what the target variable is based on the formula and the variable values for that row.
